About the project

Built on
precision.

APEX is a university-grade student ranking system. It takes a single, authoritative data source — the university records feed — and applies a deterministic C++ engine to compute CGPA, rank students, and surface the results through a clean, authenticated web interface.

Architecture

How it works

APEX is a full-stack system with a clear separation between the data layer, computation engine, and presentation. Student records are ingested from a single university source, processed by the C++ ranking engine, then served over HTTP to the frontend.

Technology

Stack

Purposefully minimal. Every component was chosen to eliminate dependency overhead while maintaining correctness and performance.

C++17 Backend

Core ranking and CGPA computation. The engine is stateless, deterministic, and runs in O(n log n) — dominated by the sort pass.

cpp-httplib HTTP layer

Single-header HTTP/HTTPS library for serving API endpoints. Handles authentication middleware, JSON serialization, and CORS.

HTML5 / CSS3 Frontend

Semantic markup and a handcrafted stylesheet. No frameworks, no build step — the browser receives exactly what was written.

Vanilla JS Interactivity

Light DOM scripting for live search, form validation, and the add-subject flow. Zero external JavaScript dependencies.

Methodology

CGPA calculation

APEX uses a credit-weighted grade point average consistent with the University Grants Commission (UGC) 10-point grading scale.

SGPA = Σ (Grade Point × Credits) / Σ Credits
CGPA = Σ (SGPA × Semester Credits) / Σ Total Credits
